## КриптоПРО

*CSP* — Cryptographic Service Provider — «поставщик криптографических услуг»

*Закрытый ключ*:
- храниться только у тебя
- можно подписать документ, расшифровать

*Открытый ключ*:
- можно раздавать всем
- можно проверить подпись

*Сертификат*
- `.cer` или `.crt`
- бинарник
- хранит открытый ключ и метаданные (ФИО владельца, срок, кем выдан, серийный номер)

Пара ключей создаются вместе и связана математически.
Голый ключ хранить обычно плохо, поэтому его упаковывают в контейнер.

*Контейнер*:
- `.key`
- бинарник (часто зашифрованный)
- содержит закрытый ключ, маски ключей, заголовки, пароли, служебную инфу
- программный контейнер находится в папке `/var/opt/cprocsp/keys/` или на флешке
- аппаратный контейнер находится на рутокене, JaCarta, eSmart и т.д.

Контейнер может стоять из нескольких файлов .key:
- header.key,    Заголовок контейнера, имя владельца, срок действия и т.д.
- masks.key,     Первая часть масок ГОСТ (защита от извлечения)
- masks2.key,    Вторая часть масок
- primary.key,   Основная часть зашифрованного закрытого ключа
- primary2.key,  Дублирующая часть (для надёжности)
- имя.000,       Папка-контейнер (внутри служебные данные)

*УЦ* — удостоверяющий центр. Организация, которой государство официально разрешило выдавать электронные подписи (УЦ ФНС, контур, таксом).
Приходишь к ним с паспортом, они проверяют и выдают тебе электронную подпись.
Выданный сертификат подписывается своим закрытым ключом.

*Корневой сертификат*
- выпускают минцифры РФ, подписывают ими сертификаты УЦ
- храниться в `/var/opt/cprocsp/certs/uroot/` или `/etc/opt/cprocsp/certs/`

*Цепочка доверия*: мой сертификат подписан УЦ, а их сертификат корневым сертификатом

*Отпечаток* (thumbprint) — уникальный идентификатор сертификата, SHA-1 или SHA-256 хеш от всего содержимого сертификата. Нужен чтобы идентифицировать сертификат.

## Подписание
1) Программа считает хеш документа
2) КриптоПро берет закрытый ключ из контейнера
3) Делает подпись (из хеша и закрытого ключа)
4) (опционально) К документу приклеивается подпись + сертификат


## Проверка
1) Получатель берет твою подпись, расшифровывает открытым ключом
2) Получает хэш файла, считает хэш документа
3) Если он совпали, значит документ не изменён


## CLI
*csptest* — работа с контейнерами и ключами (перечислить, проверить, сгенерировать, удалить)
*certmgr* — работа с сертификатами и хранилищами (установить, удалить, посмотреть)
*cryptcp* — подписать/зашифровать файл из командной строки (платная часть)
*cpverify* — проверить подпись на файле


## Шифрование
*CSR* — запрос на выдачу сертификата, содержит мета данные (ФИО, организацию и т. д.) + открытый ключ + подпись этого запроса закрытым ключом
CSR отправляется в УЦ, он выдаёт обратно сертификат

CSR бывает 2 видов:
- PKCS #10 (формат запроса на сертификат, имеет расширение `.p10` или `.csr` или `.req`)
- PKCS #7 (имеет расширение `p7c` или `.p7b`)

*PKCS* (Public-Key Cryptography Standards) — стандарты криптографии с открытым ключом




## Создание запроса
```sh
cryptcp -creatrqst -dn "CN=Иванов Иван Иванович, SNILS=123-456-789 01" -cont "\\.\HDIMAGE\abcd1234.000" request.txt
```









1.
Скопировать закрытый ключ в хранилище, где <username> - имя пользователя linux:
sudo cp -R /path/to/key/999996.000 /var/opt/cprocsp/keys/<username>/
--------

2.
Поставить «минимальные» права:
sudo chmod 600 -R /var/opt/cprocsp/keys/<username>/999996.000/
--------

3.
Узнать реальное название контейнера:
sudo /opt/cprocsp/bin/amd64/csptest -keyset -enum_cont -verifycontext -fqcn
sudo /opt/cprocsp/bin/amd64/csptest -keyset -enum_cont -fqcn -verifyc | iconv -f cp1251  
--------

4.
Ассоциировать сертификат с контейнером, сертификат попадет в пользовательское хранилище My:
sudo /opt/cprocsp/bin/amd64/certmgr -inst -file "/home/vit sol open.cer" -cont '\\.\HDIMAGE\OOO_VDELO_Solovev_28022023_1677558645 копия'
--------

5.
Установить сертификат УЦ из-под пользователя root командой:
sudo /opt/cprocsp/bin/amd64/certmgr -inst -store uroot -file /path/to/file/CA.cer
--------

6.
Проверка успешности установки закрытого ключа
sudo /opt/cprocsp/bin/amd64/certmgr --list
--------

Если выводится PrivateKey Link: Yes. Container: HDIMAGE\\999996.000\D7BB, то есть и сертификат, и закрытый ключ, а если выводится PrivateKey Link: No - связи нет, и использовать такой контейнер для подписи не удастся.
--------


Смена пароля на контейнер (снятие паролей с контейнера)
sudo /opt/cprocsp/bin/amd64/csptestf -passwd -cont '\\.\Aktiv Rutoken ECP 00 00\TestContainer' -change <новый_пароль> -passwd <старый_пароль>

В случае, если контейнеру с ключом не был задан PIN, следует воспользоваться командой: 
sudo /opt/cprocsp/bin/amd64/csptestf -passwd -cont '\\.\HDIMAGE\OOO_VDELO_Solovev_28022023_1677558645 копия' -change 'vFQHjpY586}I}=Wn'
sudo /opt/cprocsp/bin/amd64/csptestf -passwd -cont '\\.\HDIMAGE\Popova_Kseniya_Andreevna_09022024_1707464198 копия' -change '0r@G!G7YqEk1Kc-e'
-------

Экспорт контейнера и сертификата
sudo /opt/cprocsp/bin/amd64/csptest -keycopy -contsrc '\\.\HDIMAGE\OOO_VDELO_Solovev_28022023_1677558645 копия' -contdest '\\.\HDIMAGE\OOO_VDELO_Solovev_28022023_1677558645'

Установка сертификата на Тюмени
```sh
scp vdelo3.zip getlenddev:

ssh getlenddev
unzip vdelo3.zip
# имя потом понадобится
cat 25609/name.key
# избавился от кириллицы
mv \#U0412#U0414#U0435#U043b#U043e\ #U0417.cer vdelo.cer

sudo cp -R 25609 /var/opt/cprocsp/keys/root/
sudo chmod 600 -R /var/opt/cprocsp/keys/root/25609
sudo /opt/cprocsp/bin/amd64/csptest -keyset -enum_cont -verifycontext -fqcn


sudo /opt/cprocsp/bin/amd64/certmgr -inst -file "/home/stoic/vdelo.cer" -cont '\\.\HDIMAGE\13882726-7964-4c89-9cf9-de898b309402 копия'
```
